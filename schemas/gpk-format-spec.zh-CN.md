# GalEngine 数据包 (.gpk) 格式规范

## 概述

`.gpk`（GalEngine PacKage）格式是 GalEngine 游戏和补丁/DLC 的编译后二进制格式。
它将游戏素材和编译后的脚本打包为单个可分发的文件。

## 版本

当前格式版本：`1.0`

## 文件结构

```
┌─────────────────────────────────┐
│  文件头 (64 字节)                 │
├─────────────────────────────────┤
│  文件索引 (可变长度)               │
├─────────────────────────────────┤
│  元数据块 (JSON, zlib 压缩)       │
├─────────────────────────────────┤
│  脚本块 (编译后的场景数据,         │
│  zlib 压缩)                      │
├─────────────────────────────────┤
│  素材数据块                       │
│  (原始或加密的素材文件)             │
└─────────────────────────────────┘
```

## 文件头 (64 字节)

| 偏移 | 大小 | 字段           | 说明                           |
|------|------|----------------|--------------------------------|
| 0    | 4    | magic          | `0x4750474B` ("GPKG")         |
| 4    | 2    | version        | 格式版本 (1)                    |
| 6    | 2    | flags          | 位标志（见下文）                 |
| 8    | 8    | index_offset   | 文件索引的字节偏移量             |
| 16   | 8    | index_size     | 文件索引的字节大小               |
| 24   | 8    | meta_offset    | 元数据块的字节偏移量             |
| 32   | 8    | meta_size      | 元数据压缩后的大小               |
| 40   | 8    | assets_offset  | 素材数据的字节偏移量             |
| 48   | 8    | assets_size    | 素材数据的总大小                 |
| 56   | 8    | reserved       | 保留供将来使用                   |

### 标志位

| 位  | 名称         | 说明                           |
|-----|-------------|--------------------------------|
| 0   | ENCRYPTED   | 素材已加密                      |
| 1   | PATCH       | 这是一个补丁/DLC 包              |
| 2   | COMPRESSED  | 整体归档已压缩                   |
| 3-15| RESERVED    | 保留                            |

## 文件索引

文件条目数组，每个条目 128 字节：

| 偏移 | 大小 | 字段         | 说明                           |
|------|------|-------------|--------------------------------|
| 0    | 64   | filename    | UTF-8 路径（空字节填充）          |
| 64   | 8    | offset      | 素材块内的字节偏移量             |
| 72   | 8    | size        | 未压缩的文件大小                 |
| 80   | 8    | compressed  | 压缩后的大小（原始文件则为 0）     |
| 88   | 4    | crc32       | CRC32 校验和                    |
| 92   | 4    | type        | 0=原始, 1=zlib, 2=lzma          |
| 96   | 32   | reserved    | 保留                            |

## 元数据块

使用 zlib 压缩的 JSON 对象：

```json
{
  "pack_id": "com.example.mygame",
  "version": "1.0.0",
  "type": "game",
  "engine_version": "0.1.0",
  "title": "我的游戏",
  "author": "开发者",
  "scenes": ["scene_prologue", "scene_chapter1", ...],
  "languages": ["zh-CN", "en"],
  "dependencies": {
    "base_game": ">=1.0.0"
  },
  "created": "2026-05-31T12:00:00Z"
}
```

## 素材数据块

包含所有游戏素材（图片、音频、字体），按顺序拼接，可选择按文件进行单独压缩。
文件按照文件索引中列出的顺序存储。
