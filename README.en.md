## Introduction

GalEngine is an open-source visual novel game engine that enables developers to quickly create galgames, text adventures, and point-and-click puzzle games using JSON or Markdown scripts.

### Core Features

- **Dual Script Formats** — Supports both JSON (auto-generated by GUI editor) and Markdown (hand-written) script formats
- **Zero-Config Development** — Developers only need to prepare assets and scripts; no development environment setup required
- **Standalone Compilation & Packaging** — Compile game projects into standalone executables and data packs; players can download and play directly
- **Graphical Editor** — Electron + React + Monaco Editor (TypeScript rewrite v0.2.0)
- **Multilingual Support** — Developer guides and LLM prompt references in Chinese, English, and Japanese
- **Patch/DLC System** — Compile additional content into independent data packs; players can simply copy and use them
- **Dual Licensing** — Open-source (Apache 2.0) + Commercial license

## Environment Setup

### Step 1: Install Node.js

The TypeScript editor requires **Node.js 18+**.

- Download: <https://nodejs.org/>

### Step 2: Install GalEngine TypeScript Editor

```bash
# Clone the repository
git clone https://github.com/Unk1ndledAC/GalEngine.git
cd GalEngine

# Install editor dependencies
cd editor_ts
npm install
```

### Step 3: Verify Installation

```bash
# Start Electron dev mode
npm run electron:dev

# Or start Vite dev server (no Electron)
npm run dev
```

## Quick Start

### Launch the Editor

```bash
cd editor_ts
npm run electron:dev
```

The editor opens as a **standalone desktop window** (1400×900 default, minimum 900×600).

### Create a New Project

1. Click menu → **File → New Project**
2. Choose project name and location
3. The editor auto-creates the project structure and opens it

### Edit Scene Scripts

1. Click a `.json` or `.md` file in the left Explorer
2. Monaco Editor provides JSON syntax highlighting and auto-completion
3. Use the **Preview** panel to see live game preview

### Build & Package

```bash
npm run build            # Compile TypeScript + build renderer
npm run start            # Build + launch Electron (production)
npm run package          # Full packaging → NSIS installer
```

Packaging uses **electron-builder**. Output goes to `editor_ts/release/`.

## TypeScript Editor (editor_ts/)

The primary editor is a TypeScript rewrite using Electron + React + Monaco Editor, currently at **v0.2.0**.

### Tech Stack

| Component | Technology |
|-----------|------------|
| Desktop framework | Electron 42.3.0 |
| Frontend | React 18.3.1 + TypeScript 5.6 |
| Code editor | Monaco Editor (`@monaco-editor/react`) |
| State management | Zustand 5.0 |
| Build tool | Vite 6.0 |
| i18n | Custom `useTranslation` hook |

### Interface Layout

The editor uses a VSCode-style multi-panel layout:

```
┌──────────────────────────────────────────┐
│  MenuBar (File / Edit / View / Help)     │
├────┬─────────────────────────────────────┤
│    │                                      │
│ A  │        Monaco Editor (main area)    │
│ c  │                                      │
│ t  │                                      │
│ i  │                                      │
│ v  ├─────────────────────────────────────┤
│ i  │        Preview Panel (Canvas2D)      │
│ t  │                                      │
│ y  ├─────────────────────────────────────┤
│ B  │        Bottom Panel (Debug / Search)  │
│ a  ├─────────────────────────────────────┤
│ r  │  StatusBar (encoding / lang / cursor)│
└────┴─────────────────────────────────────┘
```
